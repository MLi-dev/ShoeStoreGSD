  <behavior>
    Module-level lock: `_cart_lock = asyncio.Lock()` — used for inventory check+decrement.

    All functions are synchronous EXCEPT add_item and update_quantity which are `async` (they acquire the lock).

    add_item(user_id: str, product_id: str, quantity: int) -> dict  [async]:
      - Look up product in products_db; not found → {"success": False, "code": "PRODUCT_NOT_FOUND", "message": "Product not found", "retryable": False}
      - async with _cart_lock: check product.inventory; if inventory == 0 → {"success": False, "code": "OUT_OF_STOCK", "message": "Product is out of stock", "retryable": False} (D-07)
      - If user already has an item with same product_id in cart → increment existing CartItem.quantity by quantity (D-08 merge, NOT a new line)
      - Else: create new CartItem(product_id=product_id, quantity=quantity, unit_price=product.unit_price) and append to cart
      - Create Cart if user has none yet: carts_db[user_id] = Cart(user_id=user_id)
      - Return {"success": True, "data": {"cart": _cart_to_dict(cart)}}

    update_quantity(user_id: str, product_id: str, quantity: int) -> dict  [async]:
      - Get cart from carts_db; not found → {"success": False, "code": "CART_NOT_FOUND", "message": "Cart not found", "retryable": False}
      - Find item with product_id; not found → {"success": False, "code": "ITEM_NOT_IN_CART", "message": "Item not in cart", "retryable": False}
      - If quantity <= 0: call remove_item() and return its result
      - async with _cart_lock: set item.quantity = quantity
      - Return {"success": True, "data": {"cart": _cart_to_dict(cart)}}

    remove_item(user_id: str, product_id: str) -> dict  [sync]:
      - Get cart; not found → CART_NOT_FOUND
      - Filter out the item with matching product_id; if no item removed → ITEM_NOT_IN_CART
      - Update cart.items in place
      - Return {"success": True, "data": {"cart": _cart_to_dict(cart)}}

    get_cart(user_id: str) -> dict  [sync]:
      - Return cart contents with computed total (D-09)
      - If no cart: return {"success": True, "data": {"cart": {"user_id": user_id, "items": [], "total": 0.0}}}
      - Return {"success": True, "data": {"cart": {**_cart_to_dict(cart), "total": get_cart_total(user_id)["data"]["total"]}}}

    get_cart_total(user_id: str) -> dict  [sync]:
      - Sum item.quantity * item.unit_price across all items (D-09)
      - Return {"success": True, "data": {"total": total_float}}

    clear_cart(user_id: str) -> dict  [sync]:
      - Remove cart from carts_db (or set items to []); idempotent
      - Return {"success": True, "data": {"message": "Cart cleared"}}

    Private helper _cart_to_dict(cart: Cart) -> dict:
      - Returns {"user_id": cart.user_id, "items": [{"product_id": i.product_id, "quantity": i.quantity, "unit_price": i.unit_price} for i in cart.items]}

    CartItem.variant_id: CartItem has no variant_id field in the current model. Merge behavior (D-08) is keyed on product_id only (matching the existing dataclass shape).

      - Create Order(id=str(uuid4()), user_id=user_id, items=order_items, total_amount=total, payment_method=payment_method, payment_status="pending", order_status="placed", created_at=now_iso, updated_at=now_iso)

      - Return {"success": True, "data": {"order_id": order.id, "order_status": "canceled"}}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → add_item() | Untrusted product_id and quantity from future route/agent layer |
| caller → cancel_order() | user_id MUST come from JWT session, never from request body |
| caller → request_return() | user_id MUST come from JWT session, never from request body |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Spoofing | cancel_order() | mitigate | Ownership check order.user_id == user_id enforced in service layer (D-13); cannot be bypassed by a thin router |
| T-02-08 | Spoofing | request_return() | mitigate | Same ownership check as cancel_order (D-13) |
| T-02-09 | Denial of Service | add_item() | mitigate | asyncio.Lock prevents inventory race and oversell; check-then-decrement is atomic within lock scope |
| T-02-10 | Tampering | cancel_order() | mitigate | Eligibility check on order_status before mutation prevents canceling already-shipped orders |
| T-02-11 | Elevation of Privilege | place_order() | accept | In-memory demo; no payment capture at this layer — payment mocks run in Phase 3 checkout flow |
</threat_model>
