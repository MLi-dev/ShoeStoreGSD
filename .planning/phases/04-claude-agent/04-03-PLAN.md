**POST /chat/message — AJAX endpoint (D-14, D-15):**
```python
@router.post("/chat/message")
async def chat_message(
    request: Request,
    current_user: User = Depends(get_current_user_web),
) -> JSONResponse:
    """Handle a single chat turn — run guardrails then agent loop.

    Parses JSON body {message: str}. Strips [root]: prefix when DEMO_MODE is
    True (D-14) before any further processing. Runs injection guardrail check
    (D-06). Calls agent.run() with authenticated user_id from JWT — never from
    message content (D-15). Returns JSON {reply: str}.

    Args:
        request: Current Starlette Request (JSON body).
        current_user: Authenticated User from JWT cookie.

    Returns:
        JSONResponse with {reply: str} on success.

    Raises:
        HTTPException(400): If message body is missing or empty.
        HTTPException(500): If agent loop returns a non-retryable error.
    """
    body = await request.json()
    user_message: str = body.get("message", "").strip()
    if not user_message:
        raise HTTPException(status_code=400, detail="Message required")

    # D-14: strip [root]: prefix in route handler before messages array is built.
    # Gated on DEMO_MODE so production behavior is unchanged.
    # The stripped instruction (if any) is NOT processed here — Phase 5 handles it.
    if DEMO_MODE and user_message.startswith("[root]:"):
        logger.warning(
            "Root instruction received from user_id=%s (Phase 5 feature — stripping)",
            current_user.id,
        )
        user_message = user_message[len("[root]:"):].strip()
        if not user_message:
            return JSONResponse({"reply": "Root instruction received. (Phase 5 feature — no action taken.)"})

    # D-06: Layer 1 guardrail — injection pattern check before agent sees the message.
    guard = check_message(user_message)
    if not guard["success"]:
        logger.info(
            "Injection pattern detected for user_id=%s, code=%s",
            current_user.id,
            guard.get("code"),
        )
        return JSONResponse({"reply": guard["message"]})

    # D-15: user_id comes exclusively from JWT session, never from message content.
    result = await agent.run(
        user_id=current_user.id,
        user=current_user,
        message=user_message,
    )

    if not result["success"]:
        logger.error(
            "Agent error for user_id=%s: %s",
            current_user.id,
            result.get("code"),
        )
        raise HTTPException(status_code=500, detail=result.get("message", "Agent error"))

    return JSONResponse({"reply": result["data"]["reply"]})
```

<!-- D-02: marked.js from CDN — no build step -->
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
<script>
  // Allow Enter key to submit
  document.getElementById("user-input").addEventListener("keydown", function(e) {
    if (e.key === "Enter" && !e.shiftKey) {
      e.preventDefault();
      sendMessage();
    }
  });

  function appendMessage(role, text) {
    const transcript = document.getElementById("transcript");
    const div = document.createElement("div");
    div.className = "mb-3 " + (role === "agent" ? "text-start" : "text-end");

    if (role === "agent") {
      // D-02: render agent responses as markdown via marked.js
      div.innerHTML =
        '<div class="d-inline-block bg-light border rounded p-2 text-start" style="max-width:85%">'
        + marked.parse(text)
        + '</div>';
    } else {
      // D-03: user messages rendered as plain text (escape HTML)
      const escaped = text.replace(/&/g,"&amp;").replace(/</g,"&lt;").replace(/>/g,"&gt;");
      div.innerHTML =
        '<div class="d-inline-block bg-primary text-white rounded p-2" style="max-width:85%">'
        + escaped
        + '</div>';
    }

    transcript.appendChild(div);
    div.scrollIntoView({ behavior: "smooth" });
  }

  function setLoading(loading) {
    const btn = document.getElementById("send-btn");
    const input = document.getElementById("user-input");
    btn.disabled = loading;
    input.disabled = loading;
    btn.innerHTML = loading
      ? '<span class="spinner-border spinner-border-sm" role="status"></span> Thinking...'
      : '<i class="bi bi-send"></i> Send';
  }

  async function sendMessage() {
    const input = document.getElementById("user-input");
    const message = input.value.trim();
    if (!message) return;

    appendMessage("user", message);
    input.value = "";
    setLoading(true);

    try {
      const response = await fetch("/chat/message", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ message }),

        // Session expired — redirect to login
        window.location.href = "/login?next=/chat";
        return;
      }

      if (!response.ok) {
        const err = await response.json().catch(() => ({ detail: "Unknown error" }));
        appendMessage("agent", "Sorry, something went wrong. Please try again.");
        console.error("Chat error:", err);
        return;
      }

      const data = await response.json();
      appendMessage("agent", data.reply);
    } catch (err) {
      appendMessage("agent", "Connection error. Please check your network and try again.");
      console.error("Fetch error:", err);
    } finally {
      setLoading(false);
      document.getElementById("user-input").focus();
    }
  }
</script>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /chat/message | User-controlled JSON body; message field is untrusted input |
| chat.html JS → DOM | Agent reply rendered via marked.parse() — potential XSS if marked doesn't sanitize |
| [root]: detection | Route handler must strip prefix before guardrail check; DEMO_MODE gates processing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Injection | POST /chat/message body parsing | mitigate | `body.get("message", "").strip()` — empty/missing message returns 400; guardrails.check_message() runs before agent.run(); [root]: prefix stripped before guardrail check |
| T-04-12 | Injection (XSS) | chat.html marked.parse() | accept | marked.js parses agent-generated Markdown — the agent output is LLM-generated, not raw user input; user messages use plain-text escaping (replace &lt;/&gt;/&amp;); acceptable risk for a demo app |
| T-04-13 | Elevation of Privilege | [root]: token in chat endpoint | mitigate | Stripping gated on `DEMO_MODE`; stripped content is logged at WARNING; the instruction is NOT acted on in Phase 4 (Phase 5 adds execution); user_id always comes from JWT not message |
| T-04-14 | Spoofing | Session expiry during AJAX | mitigate | 307 redirect detected by JS (`response.status === 307 || response.redirected`) — page redirects to /login, preventing unauthenticated API calls from succeeding silently |
| T-04-15 | Information Disclosure | HTTPException detail in 500 | mitigate | Agent error message is human-readable (`result.get("message")`) not a stack trace; SDK internals are never surfaced to the user |
</threat_model>
