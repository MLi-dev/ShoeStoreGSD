**_extract_text(response: Message) -> str** (private):
- Iterates response.content blocks
- Finds the first block where `block.type == "text"`
- Returns `block.text` or `""` if no text block found

**_dispatch_tool(user_id: str, tool_use: Any) -> dict** (private async):
- Looks up `tool_use.name` in `_TOOL_REGISTRY`
- If not found: returns `{"success": False, "code": "UNKNOWN_TOOL", "message": f"Unknown tool: {tool_use.name}", "retryable": False}`
- Extracts kwargs from `tool_use.input` (a dict)
- Calls `await tool_fn(user_id=user_id, **tool_use.input)`
- Returns the tool function's result dict
- Logs at DEBUG: tool name + user_id
- Do NOT raise — return failure dict on any exception

**run(user_id: str, user: User, message: str) -> dict** (public async):
Implement the agent loop per D-11 through D-13:

```python
async def run(user_id: str, user: User, message: str) -> dict:
    """Run one user turn of the agent loop.

    Appends the user message to history, then iterates up to MAX_TURNS calling
    the Anthropic API. On end_turn: extracts reply text, appends to history,
    returns success dict. On tool_use: dispatches tool, always appends tool_result
    even on exception (D-13). Caps at MAX_TURNS=10 (D-12).

    Args:
        user_id: Authenticated user ID from JWT session (never from message content, D-15).
        user: Full User dataclass for system prompt injection (D-04).
        message: Sanitized user message ([root]: already stripped by router, D-14).

    Returns:
        {"success": True, "data": {"reply": str}} on success.
        {"success": False, "code": str, "message": str, "retryable": bool} on error.
    """
    await history.append_message(user_id, {"role": "user", "content": message})
    messages = await history.get_messages(user_id)

    response: Message | None = None
    for _ in range(MAX_TURNS):
        try:
            response = await _client.messages.create(
                model="claude-opus-4-5",
                max_tokens=1024,
                system=_build_system_prompt(user),
                messages=messages,
                tools=TOOL_SCHEMAS,
            )
        except Exception as exc:
            logger.exception("Anthropic API error in agent loop")
            return {
                "success": False,
                "code": "AGENT_ERROR",
                "message": "The assistant encountered an error. Please try again.",
                "retryable": True,
            }

        if response.stop_reason == "end_turn":
            reply = _extract_text(response)
            await history.append_message(user_id, {"role": "assistant", "content": reply})
            return {"success": True, "data": {"reply": reply}}

        # stop_reason == "tool_use" — dispatch all tool calls in this response
        tool_uses = [b for b in response.content if b.type == "tool_use"]
        messages.append({"role": "assistant", "content": response.content})
        tool_results = []
        for tool_use in tool_uses:
            try:
                result = await _dispatch_tool(user_id=user_id, tool_use=tool_use)
            except Exception as exc:
                logger.exception("Tool dispatch error for %s", tool_use.name)
                result = {"error": str(exc)}
            # D-13: ALWAYS append tool_result even on exception
            tool_results.append({
                "type": "tool_result",
                "tool_use_id": tool_use.id,
                "content": str(result),
            })
        messages.append({"role": "user", "content": tool_results})

    # MAX_TURNS hard cap reached — return last partial text response
    last_reply = _extract_text(response) if response else "I reached the limit of my reasoning steps. Please try rephrasing."
    return {"success": True, "data": {"reply": last_reply}}
```

CRITICAL: The `messages` list passed to the API is built from `history.get_messages()`
which returns a copy — mutations to `messages` during the loop do NOT update stored history.
Only the final user message and final assistant reply are stored in history (via append_message).
The tool-use intermediate turns live only in the local `messages` list for this call.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chat_router → agent.run() | Route handler passes user_id from JWT, never from request body |
| agent.run() → Anthropic API | Message array must not contain raw user input that bypasses guardrails |
| Anthropic response → tool dispatch | Tool name and input come from LLM output — must be validated before dispatch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Denial of Service | agent.run() MAX_TURNS loop | mitigate | Hard cap `for _ in range(MAX_TURNS)` with break on end_turn; loop cannot run more than 10 iterations regardless of LLM behavior |
| T-04-07 | Tampering | _dispatch_tool tool name lookup | mitigate | Tool name looked up in `_TOOL_REGISTRY` dict; unknown names return UNKNOWN_TOOL failure dict without executing anything |
| T-04-08 | Information Disclosure | Anthropic API exception detail | mitigate | API exceptions are logged at WARNING/exception level but callers receive only `"The assistant encountered an error"` — no stack trace or SDK internals exposed to user |
| T-04-09 | Elevation of Privilege | user_id in run() signature | mitigate | user_id is a parameter passed by the route handler from JWT claim; the agent loop never reads user_id from the `message` string — D-15 enforced by function signature design |
| T-04-10 | Repudiation | Tool calls and failures | mitigate | Each tool dispatch logs at DEBUG level (tool name + user_id); API errors log at exception level — all actions are traceable |
</threat_model>
