Create app/lib/agent/history.py implementing:

1. Module-level state (dict + lock):
   ```python
   import asyncio
   from typing import Any

   # Keyed by user_id. Value is list of Anthropic message dicts.
   _history: dict[str, list[dict[str, Any]]] = {}
   _history_lock = asyncio.Lock()
   ```

2. Three public async functions:

   `async def append_message(user_id: str, message: dict[str, Any]) -> None`:
   - Acquires `_history_lock`
   - If user_id not in `_history`, initializes `_history[user_id] = []`
   - Appends message to the list
   - Google-style docstring

   `async def get_messages(user_id: str) -> list[dict[str, Any]]`:
   - Acquires `_history_lock`
   - Returns a COPY of the history list: `list(_history.get(user_id, []))`
   - Returning a copy prevents the caller from mutating the stored list outside the lock
   - Google-style docstring

   `async def clear_history(user_id: str) -> None`:
   - Acquires `_history_lock`
   - Sets `_history[user_id] = []`
   - Google-style docstring

3. Import block:
   ```python
   import asyncio
   from typing import Any
   ```

4. Do NOT store User objects — only message dicts (strings and Anthropic content blocks).
   Message dicts follow Anthropic SDK format: `{"role": "user"|"assistant", "content": str|list}`.

Do NOT: import from anthropic, import from app.*, add logging.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user message → guardrails | User-controlled text that may contain injection attempts |
| tool call → service layer | Tool functions must enforce ownership via user_id from session, not message |
| history dict → agent loop | Shared mutable state; concurrent requests could corrupt without locking |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | history.py `_history` dict | mitigate | `_history_lock` (asyncio.Lock) guards all reads and writes; get_messages returns a copy to prevent external mutation |
| T-04-02 | Elevation of Privilege | tools.py cancel_order/return_order | mitigate | user_id always comes from the authenticated session passed by agent.run(), never from message content; order_service enforces ownership check with UNAUTHORIZED code |
| T-04-03 | Injection | guardrails.py check_message() | mitigate | 5 compiled regex patterns catch known prompt injection phrases before message reaches agent loop; system prompt scope declaration (D-05) is the second layer |
| T-04-04 | Information Disclosure | tools.py get_product_details | accept | Product catalog is public-facing demo data; no PII exposed by this endpoint |
| T-04-05 | Elevation of Privilege | tools.py reset_password | mitigate | user_id resolved from JWT (auth session), then looked up in users_db; no user-provided email accepted; only the authenticated user's password can be reset |
</threat_model>
