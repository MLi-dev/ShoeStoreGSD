_METHOD_ALIASES: dict[str, str] = {

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| End user → chat endpoint | Authenticated user sends chat message body over HTTPS |
| Chat endpoint → root parser | Route handler extracts raw_instruction string from message |
| Root parser → FAILURE_CONFIG | Mutations applied to global in-memory config dict |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation of Privilege | POST /chat/message — root token gating | mitigate | `DEMO_MODE` guard already in place; root block only entered when `DEMO_MODE=True` AND message starts with `[root]:` — authenticated end users cannot trigger this in production mode |
| T-05-02 | Tampering | config.FAILURE_CONFIG mutations | accept | In-memory global; intentional demo behavior; resets on process restart per spec. No persistence attack surface. |
| T-05-03 | Information Disclosure | logger.warning logs raw_instruction | accept | Logs are server-side only; instruction text is operator-controlled and intentionally verbose for debugging |
| T-05-04 | Denial of Service | Malformed percentage values (e.g., "9999%") | mitigate | Parser divides raw int by 100.0 producing float; FAILURE_CONFIG values are floats read by `random.random() < prob` — values > 1.0 are treated as always-fail, same as 1.0. Clamp to [0.0, 1.0] in parser with `min(1.0, max(0.0, pct))`. |
| T-05-05 | Spoofing | Non-operator user sending [root]: prefix | mitigate | `DEMO_MODE=True` means all authenticated users can send root instructions in demo context — this is acceptable for a demo tool. Production deployment sets `DEMO_MODE=False`. Document in README. |
</threat_model>
